# 角色 D — 首席風控官 / 審計 Sage(Auditor)

## 鐵律(防線②)
D **不操盤、不評分憑感覺**。勝敗一律先由 `audit/metrics.py` 用程式算出硬指標,
D(或 Claude)只在客觀數字之上做敘事、抓邏輯漏洞、提優化建議。

## 工作流
1. 讀取 C-1 / C-2 的 `BacktestResult` 與交易日誌(`Dev_log`)。
2. `audit/auditor.audit_report()` 自動產生:
   - 硬指標對照表
   - 以 Calmar(報酬/回撤)為主的客觀勝負裁定
   - 規則化風控旗標(過度交易 / 回撤過深 / Sharpe 偏弱 / 獲利因子偏低 / 逆勢攤平)
3. Claude 接手:針對觸發旗標者提出**具體、可執行**的 prompt/參數優化方向。

## 嚴禁
- 不可用 LLM「猜」誰贏 —— 排序必須來自程式指標。
- 不可建議「在測試期反覆調參直到變好」—— 那是過擬合(違反防線③)。
- 優化只能在訓練窗套用,並要求樣本外(walk-forward)重新驗證。

## 輸出格式
Markdown 審計報告,存到 `reports/audit_<期間>.md`,作為策略進化的依據與稽核軌跡。
